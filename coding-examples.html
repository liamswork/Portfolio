SQL Database query

Using a relational database, I retrieve some data using a subquery, and ordered the results.

Read More

Objective

My objective was to query a relational database to find all drama movies and list them in order of year released (ascending).

Method

Movie genres are located within a table named "movie_genres". This table only contains movie IDs, and an associated genre ID for that movie.

The "genres" table contains the plain text description of what the genre ID actually is. Using a subquery, I filter my results to only show movies that have a genre ID equal to the "Drama" genre ID.

Social Media Button SCSS Mixins

Using multiple SCSS mixins, I create buttons that, hover, levitate and also colour depending on their class.

Read More

Objective

Expand my knowledge of SCSS by experimenting with reusable code blocks using mixins to create complex social media buttons.

Method

The first mixin takes an argument of which direction the box should glow in. In our case, I've only shown the "all" direction, but other directions exist for other elements that may use this glow scheme.

The mixin also takes a state and defaults to 'base'. Base is the buttons without any hover/focus. In scss, we can attach a second instance of the mixin for the hover version, but just pass the hover. All this state does is enlarge the glow to give it emphasis.

Above is our next step. We apply a transition and a hover state to our object that moves the item top:-10px.

Finally, below is the most complicated part of our mixin. The Colour-by-class mixin.

In our base styles, we have a colour map, with social media colours and a colour type (base, light dark)

We use this mixin to style any anchor tags of a class that matches the name in the colour map. We change the background colour to white, as the icons are originally white, making a nice transition..
